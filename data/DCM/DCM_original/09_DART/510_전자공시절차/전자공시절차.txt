  <div class="faqCont">
   <p class="txtCB">
    <b>
     1.전자문서 제출인 준비사항
    </b>
   </p>
   <dl>
    <dt>
     1-1. 전자문서 제출인 등록 신청은 어떻게 하나요?
    </dt>
    <dd>
     금융감독원 DART접수시스템 홈페이지(https://filer.fss.or.kr)에 접속하여 초기화면의 [고유번호 발급신청]을 클릭하면 신청 화면이 나타납니다.
     <br/>
     신청 후 담당자의 업무처리가 완료되면(영업일 기준 최대 익일까지) 접수홈페이지의 초기화면에서 [신청결과 조회]를 클릭하면 발급된 고유번호 등을 직접 확인할 수 있습니다. (원하시는 경우 SMS통보를 받으실 수 있습니다.)
    </dd>
    <dt>
     1-2. 고유번호만 발급받은 경우 비밀번호를 추가로 받으려면 어떻게 해야 하나요?
    </dt>
    <dd>
     외감대상법인의 고유번호를 신청한 경우(공시대상회사 등록)에는 비밀번호는 발급되지 않습니다.
     <br/>
     추후 해당 법인이 문서를 제출하고자 하는 경우 제출인으로 변경등록 해야 합니다.
     <br/>
     접수홈페이지에 접속하여 초기화면의 [고유번호 발급신청]을 클릭하면 신청 화면이 나타납니다.
     <br/>
     신청 후 담당자의 업무처리가 완료되면(영업일 기준 최대 익일까지) 접수홈페이지의 초기화면에서 [신청결과 조회]를 클릭하면 발급된 비밀번호 등을 직접 확인할 수 있습니다. (원하시는 경우 SMS통보를 받으실 수 있습니다.)
    </dd>
    <dt>
     1-3. 공시대상회사의 고유번호는 어디에서 조회하나요?
    </dt>
    <dd>
     금융감독원 DART접수시스템에 로그인하여 [게시판-회사 고유번호 조회] 메뉴를 클릭한 후 '(주)', '주식회사'를 제외한 회사명을 입력하면 고유번호를 조회할 수 있습니다.
     <br/>
     회사명이 복잡한 경우에는 중요 단어만(예:부산서부버스터미날주식회사 → 부산서부) 입력하면 그 단어가 들어있는 모든 회사를 검색할 수 있습니다. 회사명이 같은 경우에는 해당 회사를 모두 클릭하여 대 표자 및 주소 등을 확인하여야 합니다.
    </dd>
    <dt>
     1-4. 해당 업종 코드를 모릅니다.
    </dt>
    <dd>
     업종 코드를 모를 경우에는 통계청 홈페이지(http://kostat.go.kr)의 '통계분류-한국표준산업분류'에서 조회하여야 합니다. 업종코드는 5자리의 숫자로 되어 있습니다. 통계청 홈페이지에서 최초의 영문은 코드번호에 포함되지 않습니다.
    </dd>
    <dt>
     1-5. 비밀번호를 분실하였습니다. (비밀번호변경코드를 분실하였습니다.)
    </dt>
    <dd>
     ㅇ 비밀번호를 분실하였으나 비밀번호변경코드를 알고 있는 경우
     <br/>
     DART접수시스템 홈페이지(https://filer.fss.or.kr) 초기화면에서 [비밀번호등 재발급 신청]을 클릭하여 비밀번호 재발급 신청을 하면 초기화면의 [신청결과조회]에서 새로운 비밀번호를 바로 확인할 수 있습니다.
    </dd>
    <dt>
     1-6. DART접수시스템 홈페이지 접속 시 비밀번호 변경유효기간이 지났다는 메시지가 뜹니다.
    </dt>
    <dd>
     비밀번호 변경유효기간은 1년이며 이 경우 비밀번호변경코드를 알고 있으면 바로 비밀번호를 바꿀 수 있습니다. (변경하지 않아도 계속 사용할 수 있으나 보안을 위해 비밀번호 변경유효기간이 지나면 변경하실 것을 권장합니다.)
     <br/>
     주의할 사항은 비밀번호는 6자리에서 8자리로 구성되며, 숫자, 특수문자, 영문알파벳(대문자와 소문자 구별)을 이용하되, 1개 이상의 특수문자를 포함하여야 합니다. 특수문자는 컴퓨터 키보드의 아라비아숫자 위의 기호 10개를 말합니다.
     <br/>
     비밀번호변경코드는 4자리로 구성되며 특수문자와 숫자로 조합이 되어 있습니다.
    </dd>
    <dt>
     1-7. DART편집기 설치 방법을 알고 싶습니다.
    </dt>
    <dd>
     전자문서를 제출하기 위해서는 필히 DART편집기를 설치해야 합니다. (기존의 아래아한글문서나 Excel문서 등은 제출할 수 없습니다).
     <br/>
     DART접수시스템 홈페이지(https://filer.fss.or.kr) 초기화면의 [자료실]을 클릭한 후 DART제출소프트웨어 설치본을 다운로드하여 설치하시면 됩니다. DART편집기 실행시 자동으로 필요한 파일이 다운로드되어 설치되며 작성 전자서식을 클릭시 해당 문서도 다운로드되어 설치됩니다.
    </dd>
    <dt>
     1-8. DART편집기 설치파일을 다운로드 받아 설치하다보면 에러가 발생합니다.
    </dt>
    <dd>
     컴퓨터에 DART편집기가 이미 설치되어있고 다시 설치하고자 하는 경우 DART편집기가 실행 중이라면 해당 프로그램을 종료한 후 설치파일을 실행해야 합니다. 설치과정에서 실행 중인 파일은 덮어쓸 수 없기 때문입니다.
     <br/>
     위와 같은 경우가 아니라면 다운받은 설치파일이 네트워크 문제로 손상된 상태일 수도 있습니다. 다시 다운받은 후 실행하십시오.
    </dd>
    <dt>
     1-9. 인증서는 어떠한 경우에 사용하나요?
    </dt>
    <dd>
     인증서는 로그인 또는 전자서명을 할 때 사용되며 전자문서를 제출한 법인/개인의 정보를 담고 있습니다. 전자문서를 작성한 후 DART편집기에서 [제출 전송파일 생성]을 실행하면 마지막 과정 중에 '전자서명' 창이 나옵니다. 인증서 발급시에 입력했던 암호를 입력하면 전자서명이 완료됩니다.
    </dd>
    <dt>
     1-10. 인증서 발급 절차를 알고 싶습니다.
    </dt>
    <dd>
     - 공동인증서
     <br/>
     ⓛ 전자서명인증사업자나 등록대행기관(은행, 증권사 등)에서 발급
     <br/>
     ② 접수홈페이지에서 '공동인증서 로그인' 하단의 '등록/재등록'을 클릭하여 등록 신청
     <br/>
     ③ 안내에 따라 나머지 과정을 모두 진행하면 인증서 등록이 완료됩니다.
     <br/>
     <br/>
     - DART인증서
     <br/>
     ⓛ 접수홈페이지에 로그인하여 [인증서관리-DART인증서발급] 메뉴에서 각종 기재사항을 입력한 후 &lt;인증서 발급&gt;버튼을 클릭합니다.
     <br/>
     ② 안내에 따라 암호 등을 입력하면 인증서 발급이 완료됩니다.
     <br/>
     <br/>
     ※ 이때 입력한 암호는 DART편집기에서 전자서명시 필요하므로 반드시 기억하여야 합니다.
    </dd>
    <dt>
     1-11. 하나의 인증서를 한 대의 컴퓨터에 설치하여 작업을 하니 불편합니다. 인증서를 다른 컴퓨터에서도 사용하려면 어떻게 하나요?
    </dt>
    <dd>
     인증서는 여러 대의 컴퓨터에 설치하여 사용할 수 있습니다.
     <br/>
     <br/>
     ㅇ 인증서 내보내기
     <br/>
     인증서가 설치된 컴퓨터에서 DART편집기를 실행하여 [제출-인증서관리] 메뉴를 실행합니다. 인증서관리 창 중간의 &lt;고급&gt;탭을 클릭하고 하단의 &lt;인증서 내보내기&gt;버튼을 클릭하면 '-.p12' 파일로 저장할 수 있습니다. '-.p12' 파일을 다른 컴퓨터로 전달합니다.
     <br/>
     <br/>
     ㅇ 인증서 가져오기
     <br/>
     인증서를 설치할 컴퓨터에서 DART편집기를 실행하여 [제출-인증서관리] 메뉴를 실행합니다. 인증서관리 창 중간의 &lt;고급&gt;탭을 클릭하고 하단의 &lt;인증서 가져오기&gt;버튼을 클릭하여 전달받은 '-.p12' 파일을 선택하면 설치가 완료됩니다.
    </dd>
   </dl>
   <p class="txtCB">
    <b>
     2. DART편집기 (전자문서 작성)
    </b>
   </p>
   <dl>
    <dt>
     2-1. DART편집기 버전은 1.0인데 문서를 열면 2.0이라고 나옵니다.
    </dt>
    <dd>
     전자서식을 열었을때 보여지는 2.0은 문서의 서식버전입니다. DART편집기 버전과 서식버전은 서로 관계가 없습니다.
    </dd>
    <dt>
     2-2. 사용하고자 하는 서식이 없습니다
    </dt>
    <dd>
     DART편집기는 설치 후 사용하려는 서식파일을 실행하시면 인터넷을 통해 자동으로 서식을 다운로드 받습니다. 사용하시는 컴퓨터가 인터넷에 연결되었는지 확인하시기 바랍니다.
    </dd>
    <dt>
     2-3. 사업보고서를 제출할 때 감사보고서와 내부감시장치에대한감사의의견서는 제출회사가 작성해서 제출해야 하나요?
    </dt>
    <dd>
     감사보고서와 내부감시장치에대한감사의의견서는 감사인(회계법인, 감사반)이 작성하는 문서입니다. 따라서 사업보고서를 제출하는 회사는 감사보고서와 내부감시장치에대한감사의의견서를 전송파일(.DRT)로 받아서 사업보고서 본문과 함께 제출하면 됩니다.
     <br/>
     &lt;주의&gt; 전자문서파일(.dsd)로 받으면 DART편집기에서 등록을 할 수 없으니 반드시 전송파일(.DRT)로 받아야 합니다.
    </dd>
    <dt>
     2-4. 그림파일 삽입은 어떻게 하나요?
    </dt>
    <dd>
     그림파일은 jpg, jpeg, gif, bmp 확장자 형식의 파일만 삽입 가능하며 그림파일은 전자문서인 dsd안에 포함이 됩니다.
     <br/>
     증명서류 등은 스캐너를 이용하여 gif/jpg로 만들 수 있으며, 아래아 한글에서는 '도구-갈무리' 기능을 이용해 gif/jpg로 저장할 수 있습니다. MS-Word, Excel 등에서는 키보드의 PrintScreen을 사용하여 bmp로 저장한 후 DART편집기에서 [삽입-그림파일 삽입] 기능을 이용하면 파일 사이즈에 따라 자동으로 변환되어 첨부되어집니다.
    </dd>
    <dt>
     2-5. 이미지를 클립보드로 복사하여 삽입하는 방법은 없습니까?
    </dt>
    <dd>
     이미지를 클립보드에서 편집기로 직접 복사하여 삽입할 수는 없고 jpg, gif, bmp 확장자 형식의 그림파일로 만들어야만 삽입할 수 있습니다.
    </dd>
    <dt>
     2-6. 문서의 마지막 부분에 표가 있으면 그 표 이후로의 편집이 불가능합니다.
    </dt>
    <dd>
     표에 커서를 둔 후 [표-표 아래 문단 넣기] 메뉴를 실행하거나 [Shift+Enter]키를 이용하면 표 다음 부분에 빈줄이 추가되므로 내용을 추가할 수 있습니다.
    </dd>
    <dt>
     2-7. 파일형식검사시 해당 항목에 숫자, 콤마, 혹은 마이너스 기호만 삽입할 수 있다고 나옵니다.
    </dt>
    <dd>
     파란색 표에서는 빈칸에 반드시 내용을 기재해야 합니다. 이 경우 오류 메세지에 명시된 것처럼 정수 형태 (예 : -12,000)로만 기재할 수 있습니다. 기재할 내용이 없는 경우엔 '-'표시를 입력합니다.
    </dd>
    <dt>
     2-8. 파일형식검사시 '금융감독원에 문의바랍니다' 라는 메시지가 나옵니다.
    </dt>
    <dd>
     DART편집기를 사용중 위와 같은 메시지가 발생할 수 있습니다. 작성하시던 문서를 저장 후 DART편집기를 재실행하여 사용해 보십시요. 그래도 계속 같은 오류 메세지가 나오면 작성한 전자문서(*.dsd)를 이메일에 첨부하여 darthelp@fss.or.kr로 보내주십시오(이메일에 회사명, 오류 내용 등도 기재). 그러면 담당자가 수정하여 회신해드립니다.
    </dd>
    <dt>
     2-9. 문서를 인쇄할 경우 세부선택을 하고 싶습니다.
    </dt>
    <dd>
     DART편집기에서 [파일-인쇄설정] 메뉴를 실행하면 표지 및 목차 부분, 본문, 기재상의 주의 포함등으로 인쇄에 포함되는 문서부분을 원하시는 부분을 선택할 수 있습니다.
    </dd>
    <dt>
     2-10. 인쇄를 하면 표부분의 위쪽과 왼쪽 선은 잘 보이나 오른쪽의 선은 출력이 되지 않습니다.
    </dt>
    <dd>
     DART편집기는 향상된 인쇄방식을 지원합니다. 다만, 표의 넓이가 편집화면의 오른쪽 첫번째 회색선 안에 있는 경우 A4세로로 기본인쇄가능하며 두번째 회색선까지 작성되어진 표는 축소하여 A4세로로 인쇄되며 빨간석까지의 선은 축소된 형태로 A4가로인쇄가 되는 방식입니다.
     <br/>
     하지만 빨간선을 넘어선 표는 인쇄할 때 오른쪽 부분이 짤리게 됩니다.
    </dd>
   </dl>
   <p class="txtCB">
    3. DART편집기 (전송파일 만들기)
   </p>
   <dl>
    <dt>
     3-1. DART편집기의 '전송파일 생성'메뉴로 생성된 전송파일을 [제출-전송파일 관리-전송파일 내용보기] 메뉴로 열어보면 '파일이 손상되었거나 인증서 형식이 잘못되었습니다'는 메시지가 뜹니다.
    </dt>
    <dd>
     전자서명에 사용된 인증서 파일이 손상되었기 때문입니다. 기존의 인증서를 폐기하고 다시 인증서를 발급받으시기 바랍니다.
     <dt>
      3-2. 전송파일 생성의 마지막 단계인 전자서명창에서 인증서가 보이지 않습니다.
     </dt>
     <dd>
      전자서명을 하기 전에 제출인의 인증서를 먼저 발급받아야 합니다.
      <br/>
      <br/>
      - 공동인증서를 사용하셨을 경우 전자서명인증사업자나 등록대행기관(은행, 증권사)에서 발급하여 사용하셔야 합니다.
      <br/>
      - DART인증서를 사용하셨을 경우 접수홈페이지에 접속하여 [인증서관리-DART인증서발급] 메뉴에서 인증서를 발급받으시길 바랍니다.
     </dd>
     <dt>
      3-3. 전송파일 생성 과정 중 공시대상회사와 제출의무자가 동일하다는 오류가 나타납니다.
     </dt>
     <dd>
      공시대상회사란 공시서류의 작성대상이 되는 회사(예 : 유가증권시장상장법인, 코스닥시장상장법인, 코넥스시장상장법인, 외부감사대상법인 등) 입니다.
      <br/>
      제출의무자란 자본시장과 금융투자업에 관한 법률 기타 다른 법령에 따라 공시서류를 전자문서의 방법에 의하여 금융감독원에 제출해야할 의무가 있는 자입니다.
      <br/>
      사업보고서 등 대다수의 문서에서는 공시대상회사와 제출의무자가 동일합니다.
      <br/>
      반면 감사보고서의 경우 공시대상회사와 제출의무자는 서로 다릅니다. 이 경우 제출의무자는 감사를 실시한 감사인이고 공시대상회사는 감사대상회사입니다.
      <br/>
      또한 주식등의대량보유(변동)보고서나 임원·주요주주소유주식보고서의 경우 공시대상회사는 주식을 발행한 회사이고 제출의무자는 주식 등을 보유하고 있는 개인이나 법인입니다.
     </dd>
     <dt>
      3-4. 전자서명 암호(인증서 암호)를 분실하였습니다.
     </dt>
     <dd>
      - 공동인증서
      <br/>
      전자서명인증사업자나 등록대행기관(은행, 증권사 등)에서 공동인증서 메뉴를 이용하여 기존인증서를 페기하고 재발급 받아야 합니다.
      <br/>
      <br/>
      - DART인증서
      <br/>
      이 경우 접수홈페이지에 로그인한 후 [인증서관리-DART인증서폐기] 메뉴를 클릭하여 인증서를 폐기하고 다시 발급받아야 합니다.
      <br/>
      <br/>
      ※ 이때 입력한 암호는 DART편집기에서 전자서명시 필요하므로 반드시 기억하여야 합니다.
     </dd>
     <dt>
      3-5. 사업보고서 제출시 감사인으로부터 정정된 감사보고서를 받아서 제출하면 접수거부됩니다. (문서참조는 어떻게 합니까?)
     </dt>
     <dd>
      ㅇ 감사인으로부터 최종내용이 반영된 정정아닌 감사보고서를 전송파일(.DRT)로 받아서 제출하시면 됩니다.
      <br/>
      ㅇ DART편집기의 문서 참조 기능을 이용하셔도 됩니다. DART편집기에서 사업보고서의 첨부서류인 감사보고서를 선택한 후 [파일-문서 참조] 메뉴를 실행하여 감사인이 금감원에 직접 제출한 감사보고서의 접수번호 14자리를 선택 또는 입력하시면 됩니다. 접수번호는 접수홈페이지의 [제출현황조회-접수현황조회] 메뉴에서 확인하실 수 있습니다
     </dd>
    </dd>
   </dl>
   <p class="txtCB">
    4. 전자문서 제출 및 공시
   </p>
   <dl>
    <dt>
     4-1. 첨부서류는 전자문서로 제출할 때 어떤 방법으로 제출해야 하나요?
    </dt>
    <dd>
     제출인이 당사자가 되는 계약관련 서류 및 이사회의사록 등은 편집기에서 해당 첨부서류 서식을 열어 원문과 동일하게 작성(키보드로 입력)하여 제출하여야 합니다.
     <br/>
     반면에 정부 또는 공공기관 등이 발행하는 증명서류(법인등기부등본 등)는 스캐너를 통해 그림파일로 저장한 후 본문에 첨부시켜야 합니다.
     <br/>
     (자세한 방법은 아래 4-2번 답변을 참조하십시오).
    </dd>
    <dt>
     4-2. '법인등기부등본', '매매거래증빙자료'와 같이 DART편집기에서 작성할 수 없는 첨부서류는 어떻게 제출합니까?
    </dt>
    <dd>
     정부 또는 공공기관 등이 발행하는 증명서류는 스캐너를 사용하여야 합니다. 스캐닝한 후 저장할 때 그림파일의 포맷(확장자)을 gif, jpg 또는 bmp로 하여야 합니다.
     <br/>
     다음으로 편집기에서 [파일-새 문서] 메뉴로 작성하려는 첨부서류를 열고 [삽입-그림파일 삽입] 메뉴로 스캐닝된 그림파일을 문서 내에 삽입합니다. 이렇게 작업한 문서를 DART편집기에서 전송파일로 만들어 제출하면 됩니다.
     <br/>
     <br/>
     ※ 증명서류를 스캐너로 읽어들일 때는 최대한 낮은 해상도, 적은 색상으로 읽어들여야 파일크기가 작아지고 제출이 용이합니다. 참고로 그림파일을 저장할 때 16색, 256색이 사용된 이미지는 gif로 사진과 같이 많은 색상이 사용된 이미지는 jpg로 저장하시는 것이 가장 효율적입니다.
    </dd>
    <dt>
     4-3. 본문만 먼저 제출한 경우 첨부서류는 어떤 방법으로 제출해야 하나요?
    </dt>
    <dd>
     파일형식검사가 끝난 첨부서류를 작성 후 [제출-전송파일 생성] 메뉴를 이용하여 첨부서류만 등록된 상태로 전송파일 생성작업을 하시고 전송파일 생성완료 후 해당화면에서 바로 제출하기 메뉴를 이용하여 첨부서류추가 접수번호를 입력하고 제출 하실수 있습니다.
     <br/>
     <br/>
     ※ 해당 전송파일을 생성한 후 별로도 접수홈페이지에 로그인하여 [문서제출-첨부추가제출] 메뉴를 이용하여 접수하실 수도 있습니다.
    </dd>
    <dt>
     4-4. 정정서류의 작성 및 제출방법을 알고 싶습니다.
    </dt>
    <dd>
     정정서류는 이미 접수된 문서에 수정사항이 있을 때 다시 제출하는 문서입니다.
     <br/>
     DART편집기에서 이미 접수한 문서를 열고 문서상단에 녹색으로 '정정문서 작성시 이부분을 클릭하여 정오표 삽입'이란 부분을 클릭하여 정오표를 작성하고 문서 내용도 수정합니다.
     <br/>
     이 문서를 DART편집기의 [제출-전송파일 생성]을 이용하여 전송파일로 만든 뒤 해당화면에서 바로 제출하기를 선택하시면 정정대상접수번호를 입력하고 정정문서를 제출 하실수 있습니다.
     <br/>
     <br/>
     ※ 해당 전송파일을 생성한 후 별로도 접수홈페이지에 로그인하여 [문서제출-첨부추가제출] 메뉴를 이용하여 접수하실 수도 있습니다.
     <br/>
     ※ 정정문서는 반드시 접수된 문서에 한하여 제출할 수 있는 것이므로 정정하려고 하는 문서가 접수된 문서인지 확인하여야 합니다. 또한 본문이든 첨부서류든 정정할 문서만 제출하여야 하며 정정이 아닌 문서는 제출할 필요가 없습니다.
    </dd>
    <dt>
     4-5. 정정서류 제출은 언제까지 가능한가요?
    </dt>
    <dd>
     정정보고를 하는 데에는 별도의 기한이 없습니다. 정정사항이 있는 경우 가능한 한 빨리 정정서류를 제출하는 것이 좋습니다.
    </dd>
    <dt>
     4-6. 정오표를 삽입해서 정정서류를 제출했는데 '정오표 미작성'이라는 메시지가 뜨는 이유는 무엇인가요?
    </dt>
    <dd>
     정정문서를 제출할 때에는 정정이 아닌 문서를 같이 제출하면 안됩니다. 본문이든 첨부서류든 정정할 문서만 정오표를 작성하여 DART편집기를 이용하여 전송파일 생성후제출하시거나 접수홈페이지에서 [문서제출-정정서류 제출] 메뉴로 제출해야 합니다.
    </dd>
   </dl>
   <p class="txtCB">
    5 .감사인 관련
   </p>
   <dl>
    <dt>
     5-1 '제출(접수) 가능일 이전에 제출되었습니다. 제출(접수) 가능일을 확인하여 다시 제출하여 주십시오. (제출/접수 가능일 : 200XXXXX 이후 2주이내)'라는 사유로 감사보고서가 접수거부되었습니다. 접수거부된 이유는 무엇입니까?
    </dt>
    <dd>
     감사보고서는 「주식회사의외부감사에관한법률」에서 정기총회 종료 후 2주일 이내에 제출하도록 되어 있습니다. 따라서 정기총회 전에 감사보고서를 제출하면 접수가 거부됩니다.
    </dd>
    <dt>
     5-2. 감사보고서 현금흐름표에서 계정과목 및 계산이 정확한데도 재무제표 검사시 오류가 발생합니다.
    </dt>
    <dd>
     현금흐름표의 경우 단위서식으로 직접법과 간접법 2가지가 있으며 각 서식별로 '영업활동으로인한현금흐름'의 계정과목 및 계산식이 서로 다릅니다. 그러므로 해당 서식이 정확히 삽입되었는지 확인하십시오.
     <br/>
     <br/>
     ※ 이익잉여금처분계산서와 결손금처리계산서도 단위서식별로 계정과목과 계산식이 다르니 오류가 발생하는 경우 서식이 정확히 삽입되었는지 확인하시기 바랍니다.
    </dd>
    <dt>
     5-3. 기업회계기준이 개정되어 감사보고서 재무제표를 당기분만 작성하고자 합니다. 기본으로 제공되는 비교식 서식에서 전기를 어떻게 삭제합니까?
    </dt>
    <dd>
     커서를 재무제표의 전기에 위치시킨 후 [표-열 지우기] 메뉴를 실행합니다.
     <br/>
     <br/>
     ※ 다음 연도에 다시 비교식이 되는 경우 [재무제표-전기로 금액 밀기] 메뉴를 실행합니다.
    </dd>
    <dt>
     5-4. 어떤 법인의 감사보고서에 기업개황자료를 첨부합니까?
    </dt>
    <dd>
     유가증권/ 코스닥/코넥스시장상장법인이 아닌 외감법인의 감사보고서에 첨부합니다.
    </dd>
    <dt>
     5-5. 회계감사준칙이 개정되어 감사보고서에 수정재무제표를 작성하지 않는 것으로 알고 있습니다. 전자문서에도 수정재무제표를 작성하지 않아도 됩니까?
    </dt>
    <dd>
     수정재무제표를 작성하지 않습니다.
     <br/>
     ※ 수정재무제표는 대차대조표와 손익계산서에 대하여 작성하되 수정사항이 발생한 서식만 작성합니다.
    </dd>
    <dt>
     5-6. 회사에 감사보고서를 전송파일(.DRT)로 전달하였는데 회사에서 감사보고서를 전자문서에 첨부하는 과정에서 오류가 발생한다고 합니다. 전송파일(.DRT)에오류가 없는 것을 확인하고 전달하였는데도 오류가 발생한 원인은 무엇입니까?
    </dt>
    <dd>
     회사에 전달하는 감사보고서 전송파일에는 회사가 첨부하고자 하는 문서만이 있어야 합니다.
     <br/>
     예 : 회사에서 감사보고서만을 첨부하는데 전송파일(.DRT)에 기업개황자료가 함께 있는 경우 오류발생
     <br/>
     <br/>
     ※ 전송파일에 포함된 문서는 DART편집기에서 [제출-전송파일 관리-전송파일 내용보기] 메뉴를 실행하여 확인할 수 있습니다.
    </dd>
    <dt>
     5-7. '작성하신 기업개황자료의 기재 내용에 오류가 있습니다(예:업종코드불일치).'라는 사유로 감사보고서(기업개황자료)가 접수거부되었습니다. 업종코드를 찾아 정확히 기재하였는데 접수가 거부된 이유는 무엇입니까?
    </dt>
    <dd>
     개정전 업종코드를 기재한 감사보고서(기업개황자료)를 제출하면 접수가 거부됩니다.
						최신 업종코드를 통계청 홈페이지(http://kostat.go.kr)의 '통계분류-한국표준산업분류'에서 조회하여 기재하시기 바랍니다.
    </dd>
    <dt>
     5-8. 감사종료보고서를 제출했는데 배포공시홈페이지에 공시가 되지 않습니다.
    </dt>
    <dd>
     감사종료보고서는 법규상 공시서류가 아니므로 공시가 되지 않습니다.
    </dd>
    <dt>
     5-9. 감사계약체결보고서는 감사대상회사별로 작성하는 것입니까? 그리고 제출일은 언제까지입니까?
    </dt>
    <dd>
     감사계약체결보고서는 감사대상회사별로 각각 작성하여 각각 전송파일을 만들어서 제출하여야 하며 감사계약체결후 2주일이내에 제출하여야 합니다.
    </dd>
    <dt>
     5-10. 감사계약체결보고서 제출시 '공시대상회사와 제출의무자가 동일하다'는 사유로 접수거부됩니다.
    </dt>
    <dd>
     감사계약체결보고서를 전송파일로 만들때 제출의무자는 감사인의 고유번호를 기재하고, 공시대상회사는 감사대상회사의 고유번호를 기재하여야 합니다.
     <br/>
     주의할 사항은 감사대상회사의 법인등록번호를 잘못 기재하여 제출할 경우 오류로 인하여 접수거부된다는 것입니다.
     <br/>
     그리고 감사계약체결보고서는 배포공시홈페이지에서 열람이 안되는 보고서입니다.
    </dd>
   </dl>
